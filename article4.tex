\Abstract{ We present the \proglang{R} package \pkg{exdqlm} as a tool for dynamic
quantile regression. The main focus of the package is to provide a framework for Bayesian inference and forecasting of flexible dynamic quantile linear models (exDQLMs). exDQLMs utilize a new family of error distributions for parametric quantile regression, the extended asymmetric Laplace (exAL), a generalization of the asymmetric Laplace (AL) which is commonly used in parametric quantile regression. Estimation of a dynamic quantile linear model, which utilizes the AL, is included in the package as a special case. Non-time-varying quantile regression models are also a special case of our methods. Further, routines for estimation of a nonlinear relationship between the response and a given input variable at a specified quantile via a transfer function model are available. The software provides exact posterior inference via Markov chain Monte Carlo (MCMC) and fast approximate posterior inference via Laplace delta variational Bayes (LDVB). While MCMC provides sample-based exploration of the posterior distribution, the approximations obtained using the LDVB algorithm enable fast inference for long time series at a fraction of the memory and computational costs of the MCMC. A routine for forecasting the dynamic quantile is available in the package, as well as several quantitative and visual diagnostics for model evaluation. We illustrate the implementation of the functions and algorithms in the \pkg{exdqlm} package with a step-by-step guide for the analysis of several real data sets. }

\Keywords{dynamic quantile regression, dynamic models, asymmetric Laplace, variational Bayes, coordinate ascent variational inference, quantile regression}

\section{Introduction}

The purpose of the \proglang{R} \citep{R} package \pkg{exdqlm} is to provide an accessible and comprehensive tool for flexible Bayesian estimation of a single dynamic quantile. Quantile regression is an area that has gained popularity in the non-time-varying setting as a robust alternative to traditional mean-centric approaches. However, time-varying methods and software implementations remain limited. In a Bayesian setting, parametric quantile regression models are almost exclusively based on the asymmetric Laplace (AL) likelihood, as it corresponds to the check loss function that is minimized in traditional quantile regression problems \citep{yu2001bayesian,koenkerquantile}. A mixture representation of the AL provides computational advantages for the exploration of the posterior distribution of the model parameters. This has contributed to its popularity. However the AL is known to have several shortcomings in terms of flexibility. Very general error distributions for quantile regression have been considered extensively in the Bayesian non-parametric literature \citep{kottas2009bayesian,reich2009flexible,taddy2010bayesian}, but the computational cost of these methods do not scale well to dynamic models for long times series. \cite{yan2017new} present a parametric extension of the AL, the extended asymmetric Laplace (exAL), which overcomes the restrictive aspects of the distribution while preserving its computational advantages. \cite{me} utilize the new family of error distributions presented in \cite{yan2017new} to develop a flexible dynamic quantile linear model (exDQLM). This is the model implemented in the package \pkg{exdqlm}.

Dynamic quantile models typically consist of a set of variables that summarize the properties of the quantile of interest and a structure that defines the evolution of these variables, which together represent the behavior of a quantile over time. Our package \pkg{exdqlm} is a powerful and versatile tool for dynamic quantile modeling. The model form is robust enough to estimate any quantile regardless of non-standard features in the data distribution (i.e., skewness or extreme observations). Further, relevant features such as long-term trends or seasonality are easily incorporated into the evolution of the quantile. The package includes both a Markov chain Monte Carlo (MCMC) algorithm for exact posterior inference and a Laplace delta variational Bayes (LDVB) algorithm for fast, yet accurate approximate estimation. A legacy importance sampling variational Bayes (ISVB) routine is retained in the package for backward compatibility, however in this paper we focus on LDVB as the primary approximation. The LDVB routine makes our methods accessible to applications with very large datasets and supports quick model exploration and selection. The exDQLM framework facilitates $k$-step-ahead forecasting for the estimated quantile, also implemented in the package. Further, a transfer function extension is included in the software as a straightforward method to quantify a nonlinear relationship between a response and a given input variable at a specified quantile. Several visual and quantitative model diagnostics are provided to support model examination. Finally, MCMC and LDVB algorithms for static quantile regression using the exAL error distribution are also included in the package. The package is protected under the MIT License, and is available from the Comprehensive R Archive Network (CRAN) at \url{https://CRAN.R-project.org/package=exdqlm}.

Well-developed software implementations of non-time-varying quantile regression models which utilize the AL are available in the Bayesian \proglang{R} package \pkg{bayesQR} \citep{bayesQR} and, equivalently, a classical version, which utilizes the check-loss function \citep{koenkerquantile}, is available in the package \pkg{quantreg} \citep{quantreg}. As a result of the AL being a special case of the exAL, the methods used in the Bayesian package \pkg{bayesQR} are also a special case of our exDQLM implemented in \pkg{exdqlm}. Further, there are many established packages for dynamic modeling including \pkg{dlm} \citep{dlm} and \pkg{dynr} \citep{dynr}, however, implementations of dynamic quantile modeling are much more limited. To our knowledge, the package \pkg{quantreg} includes a routine which allows for a time-evolving structure in the regression coefficients (a special case of the routines in our package), but this is the extent of the software available for dynamic quantile regression.

In the remainder of this paper we introduce the package as follows. In Section \ref{sec:exdqlm}, we describe the modeling framework, including the algorithms used for efficient posterior inference and our methods for including nonlinear relationships. We also discuss static quantile regression as a special case of the framework. In Section \ref{sec:diags} we detail the model diagnostics and forecasting method implemented in the package. Then, in Section \ref{sec:exs}, we walk through four step-by-step examples on how to use the package. Finally, in Section \ref{sec:summary}, we present a summary of the package capabilities.

\section{Extended dynamic quantile linear models}
\label{sec:exdqlm}

Consider a set of scalar time-evolving responses, $y_t$, for times $t=1,\dots,T$. For each $t$, an extended dynamic quantile linear model (exDQLM) for inference on a single $p_0$-th quantile is defined by
\begin{align}
\label{EQ:exDLM}
& \text{Observation equation:} & y_t &= {\bf F}'_t \boldsymbol{\theta}_t + \epsilon_t,  & \epsilon_t & \sim \text{exAL}_{p_0}(0,\sigma,\gamma) \\
\label{EQ:exDLMevo}
& \text{System equation:} & \boldsymbol{\theta}_t &= {\bf G}_t \boldsymbol{\theta}_{t-1} + \boldsymbol{\omega}_t, & \boldsymbol{\omega}_t  & \sim \text{N}({\bf 0},{\bf W}_t).
\end{align}
Here ${\bf F}_t$ is the $q \times 1$ regression vector of the covariates corresponding to the $q \times 1$ parameter vector $\boldsymbol{\theta}_t$ at time $t$, ${\bf G}_t$ is the $q\times q$-dimensional evolution matrix defining the structure of the parameter vector evolution in time, and the normal distribution according to which the system error $\boldsymbol{\omega}_t$ evolves is $q$-variate. The observational errors $\epsilon_t$ of the exDQLM follow an exAL distribution for fixed quantile $p_0$, whose density is denoted as $\text{exAL}_{p_0}(\cdot)$, and it is such that $\int_{-\infty}^0 \text{exAL}_{p_0}(\epsilon_t |0,\sigma,\gamma) \mathrm{d}\epsilon_t = p_0$. Thus, the observation equation in Equation (\ref{EQ:exDLM}) implies that ${\bf F}'_t \boldsymbol{\theta}_t$ corresponds to the $p_0$-quantile of $y_t$,
even if this is not made explicit in the notation above. Table \ref{table:priors} shows the function parameters that define the structure of the dynamic quantile, specifically ${\bf F}_t$ and ${\bf G}_t$ for $t=1,\ldots,T$.

The exAL distribution is constructed from an extension of the location-scale mixture representation of the AL \citep{kozumi2011gibbs}, details of which can be found in \cite{yan2017new}. This results in a mixture representation of $\text{exAL}_{p_0}(0,\sigma,\gamma)$ as follows:
\begin{equation}
\label{EQ:exALmix}
\int\int_{\mathbb{R}^+ \times \mathbb{R}^+} \text{N}(y_t |  C(p,\gamma) \sigma |\gamma| s_t + A(p) v_t, \sigma B(p) v_t) \text{Exp}(v_t | \sigma) \text{N}^+(s_t| 0,1) \mathrm{d}v_t \mathrm{d}s_t.
\end{equation}
Here, $p = p(p_0,\gamma) = I(\gamma <0) + \{[p_0 - I(\gamma <0)]/g(\gamma) \}$ where $g(\gamma) = 2\Phi(-|\gamma|)\text{exp}(\gamma^2/2)$, and $\Phi(\cdot)$ denotes the standard normal CDF. $A(p)$, $B(p)$, $C(p,\gamma)$ are the functions of $p$ and $\gamma$: $A(p) = \frac{1-2p}{p(1-p)}$, $B(p) = \frac{2}{p(1-p)}$, $C(p,\gamma) = [I(\gamma>0) -p]^{-1}$. Lastly, $\text{Exp}(v | \sigma)$ denotes the exponential distribution with mean $\sigma$, and $\text{N}^+(s_t| 0,1)$ denotes a normal distribution truncated to the positive reals with mean $0$ and variance $1$. This mixture representation of the exAL can be exploited to rewrite the exDQLM as the following hierarchical model for $t=1,\dots,T$:
\begin{align}
\label{EQ:exDQLMobs}
y_t | \boldsymbol{\theta}_t,\sigma,\gamma,v_t,s_t  & \sim \text{N}(y_t | {\bf F}_t' \boldsymbol{\theta}_t + C(p,\gamma) \sigma |\gamma| s_t + A(p) v_t,\sigma B(p) v_t) \\
v_t, s_t | \sigma & \sim \text{Exp}(v_t | \sigma) N^+(s_t |0,1) \\
\label{EQ:exDQLMevo}
\boldsymbol{\theta}_t | \boldsymbol{\theta}_{t-1},{\bf W}_t & \sim  \text{N}({\bf G}_t \boldsymbol{\theta}_{t-1},{\bf W}_t).
\end{align}
This hierarchical form is leveraged for the posterior inference implemented within the \pkg{exdqlm} package.

Under a Bayesian formulation and given $p_0$, we specify priors for the initial state vector $\boldsymbol{\theta}_0$, scale parameter $\sigma$ and skewness parameter $\gamma$. For $\boldsymbol{\theta}_0$, we assume a $q$-variate conjugate normal prior $\boldsymbol{\theta}_0 \sim \text{N}({\bf m}_0,{\bf C}_0)$. For $\sigma$, we assume a conjugate inverse-gamma prior denoted as $\text{IG}(a_{\sigma},b_{\sigma})$. In many cases a strong prior on $\sigma$ is necessary to guarantee fast posterior convergence and estimation. This is illustrated in the examples of Section \ref{sec:exs}. The parameter $\gamma$ has bounded support over the interval $(L,U)$ where $L$ is the negative root of $g(\gamma) = 1-p_0$ and $U$ is the positive root of $g(\gamma) = p_0$ \citep{yan2017new}. In contrast to the flat prior suggested by \cite{yan2017new}, we find that using a proper prior on the skewness parameter promotes reliable posterior inference. Thus, we place a Student-t distribution truncated to the interval $(L,U)$ as the prior for $\gamma$, i.e. $\gamma \sim \text{t}_{(L,U)}(m_\gamma,s_\gamma)$ with $\nu_\gamma$ degrees of freedom. The prior parameters used in the \pkg{exdqlm} functions, their corresponding mathematical symbols, and default values are shown in Table \ref{table:priors}.

\subsection{Posterior estimation}

The hierarchical representation of the exDQLM facilitates posterior simulation using Markov Chain Monte Carlo (MCMC) algorithms. This is implemented in the function \code{exdqlmMCMC()}. Gibbs sampling steps are used to update the sets of latent variables $s_t$ and $v_t$ for $t=1,\dots,T$. The dynamic regression coefficients are sampled using a forward filtering backwards sampling algorithm \citep{carter1994gibbs,fruhwirth1994data}. The scale and skewness parameters are then updated separately from the latent-state blocks. By default, the package uses a slice-sampling update for the $(\sigma,\gamma)$ block, drawing $\sigma$ from its full conditional distribution and updating $\gamma$ with a bounded univariate slice sampler. Alternatively, the package also provides joint random-walk Metropolis-Hastings (MH) kernels on transformed coordinates, including a Laplace-informed proposal covariance that can be refreshed during burn-in. When desired, the chain can be warm-started from either the LDVB or legacy ISVB approximation; this initialization is separate from the subsequent MCMC proposal kernel. An example of this functionality can be found in Section \ref{sec:ex1}. We perform a total of $N_{BURN} + N_{MCMC}$ MCMC iterations, discarding the first $N_{BURN}$ and retaining the remaining $N_{MCMC}$ posterior samples. The object created by \code{exdqlmMCMC()} includes posterior samples of all parameters ($\boldsymbol{\theta}_{1:T},v_{1:T},s_{1:T},\gamma,\sigma$), filtered and smoothed summaries for the state vector $\boldsymbol{\theta}_{1:T}$, and diagnostics for the proposal kernel used in the $(\sigma,\gamma)$ update.

For purposes of model selection or large time series data sets, posterior sampling with the MCMC algorithm can be computationally demanding. To this end, we include in the package a Laplace delta variational Bayes (LDVB) routine, \code{exdqlmLDVB()}, for fast and accurate approximate posterior estimation. The variational Bayes (VB) framework approximates the posterior distribution with partitioned variational distributions \citep{ostwald2014tutorial,beal2003variational}. Under a mean-field factorization, this induces a blockwise coordinate-ascent variational inference scheme. For the exDQLM, the blocks associated with the dynamic coefficients and latent variables admit recognizable closed-form updates, while the joint variational distribution of $\sigma$ and $\gamma$ is non-conjugate. Following the non-conjugate VB strategy of \cite{wang2013nonconjugatevb}, we approximate this block with a Laplace approximation on transformed coordinates and then use a second-order Delta method to evaluate the expectations of functions of $(\sigma,\gamma)$ needed by the remaining variational updates. Thus, the mean-field factorization induces the blockwise CAVI updates, while the non-conjugate $(\sigma,\gamma)$ block is handled through the Laplace-Delta approximation. The dynamic regression coefficients are updated within each iteration using a forward filtering backwards smoothing algorithm. Convergence is assessed jointly through the state, scale, skewness, and ELBO diagnostics. After the algorithm reaches convergence with tolerance $\epsilon_{tol}$, $N_{SAMP}$ samples are drawn from the variational distributions, resulting in an approximate sample of the posterior distribution. The output of \code{exdqlmLDVB()} includes the approximate posterior samples as well as the parameters of the variational distributions for all parameters ($\boldsymbol{\theta}_{1:T},v_{1:T},s_{1:T},\gamma,\sigma$). For the $(\gamma,\sigma)$ block, the return also contains the transformed mode, approximate covariance matrix, derived moments, and ELBO terms associated with the Laplace-Delta approximation. The package also retains a legacy importance-sampling VB routine, \code{exdqlmISVB()}, but in this paper we restrict attention to MCMC and LDVB as the primary inferential tools.

\subsection{Specification of the evolution covariance via discount factors}
\label{sec:dfs}

Both estimation algorithms, \code{exdqlmMCMC} and \code{exdqlmLDVB}, utilize discount factors to specify the time-evolving covariance matrices ${\bf W}_t$ introduced in Equation \eqref{EQ:exDLMevo} \citep{west1985dynamic}. More precisely, we define the evolution variance matrix ${\bf W}_t = \frac{1-\delta}{\delta} {\bf G}_t {\bf C}_{t-1} {\bf G}_t'$ where ${\bf C}_{t-1}$ denotes the posterior variance of the state vector ${\boldsymbol\theta_{t-1}}$ at time $t-1$ and $\delta$ is a discount factor such that $0 < \delta \leq 1$. The discount factor $\delta$ can be specified by the user via function parameter \code{df}. Selection of discount factors is typically done by optimizing a model-checking criterion. We suggest using the KL divergence or posterior predictive loss criterion discussed in Section \ref{sec:diags}. A brief example of this method for discount factor selection can be found in Section \ref{sec:ex2}.

Discounting by component of the state-space model is also a functionality of our estimation algorithms. Suppose the state vector is comprised of $h$ components $\boldsymbol{\theta}_{it}$ of dimension $q_i$ for $i=1,\dots,h$ such that $\boldsymbol{\theta}_{t}' = (\boldsymbol{\theta}_{1t}',\dots,\boldsymbol{\theta}_{ht})'$ and $\sum_{i=1}^{h} q_i = q$. If ${\bf W}_{it}$ denotes the $i^{th}$ component of the evolution covariance matrix such that ${\bf W}_t = \text{blockdiag}\{ {\bf W}_{1t},\dots,{\bf W}_{ht}\}$ and $\delta_i$ denotes a discount factor for the $i^{th}$ component, we can define the evolution variance matrix by component as ${\bf W}_{it} = \frac{1-\delta_i}{\delta_i} {\bf G}_{it} {\bf C}_{i,t-1} {\bf G}_{it}'$. Here ${\bf G}_{it}$ and ${\bf C}_{i,t-1}$ denote the $i^{th}$ components of ${\bf G}_t$ and ${\bf C}_{t-1}$, respectively. The function parameters \code{df} and \code{dim.df} allow users to specify a set of component discount factors $(\delta_1,\dots,\delta_h)$ and their dimensions $(q_1,\dots,q_h)$, respectively, within the state-space model. Examples of this feature can be found in Section \ref{sec:exs}. For a full review of discount factors including discounting by component, see \cite{west1985dynamic}.

\subsection{Transfer function model}
\label{sec:tf}

Quantifying the relationship between an input and a quantile of a response is a non-trivial task, particularly when the relationship is nonlinear. In mean-centric dynamic modeling, transfer functions are a straightforward method to incorporate variables which measure the combined effect of current and past regression effects \citep{west1985dynamic}. An extension of the transfer function approach to the dynamic quantile modeling setting is implemented within the \pkg{exdqlm} package, particularly for the LDVB algorithm.

For $t=1,\dots,T$ and univariate input, $X_t$, a transfer function exDQLM with exponential decay is defined by
\begin{align}
\label{EQ:begin_transfn}
y_t | \boldsymbol{\theta}_t,\gamma,\sigma  & \sim \text{exAL}_{p_0}({\bf F}'_t \boldsymbol{\theta}_t +  \zeta_{t},\sigma,\gamma) \\
\boldsymbol{\theta}_t | \boldsymbol{\theta}_{t-1},{\bf W}_t  & \sim  \text{N}({\bf G}_t \boldsymbol{\theta}_{t-1},{\bf W}_t) \\
\label{EQ:transfn_effect}
\zeta_{t}|\zeta_{t-1},\psi_{t-1},\omega_{t} & \sim \text{N}(\lambda \zeta_{t-1} + X_{t}\psi_{t-1},\omega_{t}) \\
\label{EQ:end_transfn}
\psi_{t}|\psi_{t-1},\nu_{t} & \sim \text{N}(\psi_{t-1},\nu_{t}).
\end{align}
Here $\zeta_{t}$ captures the effect of the current and past regression effects on the quantile at time $t$. Alternatively, the parameter $\psi_{t}$ determines the instantaneous effect $X_t$ has on the quantile. The parameter $\lambda\in[0,1]$ represents the memory of the regression effect up to time $t$. This effect decays at an exponential rate, reducing by a factor of $\lambda$ at every time step. More explicitly, the effect of $X_t$ on the quantile at time $t+k$ is $\lambda^k \psi_{t-1}X_t$. Thus, the series $k_t \geq \frac{\log(\epsilon) - \log(|\psi_{t-1}X_t|)}{\log(\lambda)}$ represents a lower bound for the number of time steps until the effect of $X_t$ is less than or equal to a fixed $\epsilon$. The median value of $k_t$ corresponding to $\epsilon = 1\mathrm{e}{-3}$ is included as an output of our transfer function routine.

Conditional on a fixed $\lambda$, say $\tilde\lambda$, we augment the LDVB algorithm to incorporate the transfer function structure by simply replacing ${\bf F}_t, {\boldsymbol{\theta}}_t, {\bf G}_t,$ and ${\bf W}_t$ in Equations \eqref{EQ:exDLM}-\eqref{EQ:exDLMevo} with $\tilde{\bf F}'_t = ({\bf F}'_t, 1,0)$, $\tilde{\boldsymbol{\theta}}'_t = ({\boldsymbol{\theta}}'_t,\zeta_{t},\psi_{t})$, $\tilde{\bf G}_t = \text{blockdiag}\left\{{\bf G}_t,\left(\begin{smallmatrix}
\tilde\lambda & X_{t} \\ 0 & 1 \end{smallmatrix} \right)\right\}$, and  $\tilde{\bf W}_t = \text{blockdiag}\left\{{\bf W}_t,\left(\begin{smallmatrix} \omega_t & 0 \\ 0 & \nu_t \end{smallmatrix} \right)\right\}$, respectively. This augmentation is implemented in the function \code{transfn_exdqlmLDVB()}, an extension of the routine found in \code{exdqlmLDVB()}.

Similar to the specification of $\bf{W}_t$ discussed in Section \ref{sec:dfs}, discount factors $(\delta_\zeta,\delta_\psi)$ are utilized to specify $\left(\begin{smallmatrix} \omega_t & 0 \\ 0 & \nu_t \end{smallmatrix} \right)$. A bivariate normal conjugate prior on $(\zeta_{0},\psi_{0})' \sim N({\bf m}_0^{tf},{\bf C}_0^{tf})$ completes the transfer function model extension. \subsection{Special cases, including static quantile regression}
\label{sec:special}

The first special case of our exDQLM arises from the fact that the exAL is an extension of the AL \citep{yan2017new}. When $\gamma = 0$, the
observational error $\epsilon_t$ in Equation \eqref{EQ:exDLM} reduces such that $\epsilon_t \sim \text{exAL}_{p_0}(\epsilon_t |0,\sigma,\gamma) = \text{AL}_{p_0}(\epsilon_t |0,\sigma)$. This special case with observational errors distributed independently from an AL is the dynamic quantile linear model (DQLM) presented in \cite{gonccalves2017dynamic}. The DQLM can be implemented with our package using the parameter \code{dqlm.ind = TRUE}.

Non-time-varying quantile regression is also a special case of our methodology. More specifically, when discount factors are set to 1, the evolution covariance ${\bf W}_t$ reduces to ${\bf 0}$ resulting in non-time-varying parameters \citep{west1985dynamic}. Thus, the model reduces to the static exAL regression methodology of \cite{yan2017new}. This model can be implemented using our functions \code{exal_static_mcmc()} or \code{exal_static_LDVB()}, which implement MCMC and LDVB algorithms, respectively.

The final special case is the intersection of the previous two. A static quantile regression model with observational errors distributed according to an AL corresponds to Bayesian linear quantile regression under the AL working likelihood, first presented in \cite{yu2001bayesian}. This methodology, utilized to create the Bayesian package for quantile regression \pkg{bayesQR}, can also be implemented with our package using functions \code{exal_static_mcmc()} or \code{exal_static_LDVB()} with parameter \code{dqlm.ind = TRUE}, which acts as a convenience flag fixing $\gamma = 0$.

Examples of these special cases can be found in Section 4. A static quantile regression example specifically can be found in Section \ref{sec:ex4static}. For further details on the methodology outlined in this section, see \cite{me}.

\section{Model Diagnostics and Forecasting}
\label{sec:diags}

To evaluate the quantile inference and predictive performance of the exDQLM, several quantitative and visual model diagnostics are included in the package.

The one-step-ahead predictive distribution function introduced by \cite{rosenblatt1952remarks} is a useful diagnostic for models in which the marginal forecast distributions are unrecognizable or difficult to compute \citep{huerta2003time,prado2006multivariate}. For the exDQLM, this distribution is defined as
\begin{align}
u_t = \text{Pr}(Y_t \leq y_t |y_{1:t-1},v_{1:T},s_{1:T},\sigma,\gamma), % \Phi(y_t| y_{1:t-1},v_{1:T}},s_{1:T},\sigma,\gamma) =
\end{align}
with $y_{1:t-1} =y_1,\dots,y_{t-1}$, $v_{1:T}= (v_1,\dots,v_T)$ and $s_{1:T}= (s_1,\dots,s_T)$. Here $u_t$ defines an independent sequence which is uniformly distributed on the interval $(0,1)$ \citep{rosenblatt1952remarks}. Conditional on $\{v_{1:T},s_{1:T},\sigma,\gamma\}$, the forecast distribution of $y_t$ within the exDQLM is normally distributed. Therefore, conditional on MAP estimates $\{\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma\}$ we estimate the sequence as
\begin{align}
\hat{u}_t = \Phi(y_t| y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma)
\end{align}
where $\Phi$ denotes the normal CDF.

Using this estimated sequence $\{\hat u_t\}$, we can assess the model performance in several ways. First, we visually examine the correlation of the estimated sequence via an ACF plot. Second, by transforming the values with a standard normal inverse CDF, we compare their distributional shape to that of a standard normal with a normal QQ-plot. To quantify the divergence of this transformed sequence from normality we compute the KL divergence \citep{kullback1951information}, $\text{KL}(h,\phi) = \int_{-\infty}^{\infty} h(x)\log \frac{h(x)}{\phi(x)}\mathrm{d}x$, where $h$ denotes the numerically approximated density of our transformed sample and $\phi$ is the standard normal density. A smaller KL divergence suggests a superior model fit. Lastly, the MAP standardized forecast errors,
\begin{align}
\frac{y_t-\E[y_t | y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma]}{\sqrt{\text{Var}(y_t|y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma)}}
\end{align}
can also be used as an additional graphical aid in examining the distribution at specific time points. These MAP standardized forecast errors are included in the output of the \code{exdqlmMCMC} and \code{exdqlmLDVB} functions.

For a final diagnostic, we include \cite{gelfand1998model} posterior predictive loss criterion (pplc) with check loss function $\rho_{p_0}(x) = x[p_0-I(x<0)]$. That is,
\begin{align}
\text{pplc} = \sum_t \E[\rho_{p_0} (y_t^{obs}-y^{rep}_t)|y_{1:T}],
\end{align}
where the expectation is with respect to the posterior predictive distribution of $y_t$, $p(y^{rep}_t| y_{1:T})$. This distribution is used to generate a replicate dataset and is the expected value of the observation equation \eqref{EQ:exDLM} with respect to the posterior distributions of the parameters. Again, a smaller pplc value suggest a superior model fit. Samples from the posterior replicate distributions are included in the output of the \code{exdqlmMCMC()} and \code{exdqlmLDVB()} functions.

The function \code{exdqlmDiagnostics()} implements the model checking criteria discussed in this section. When an output from either \code{exdqlmMCMC()} and \code{exdqlmLDVB()} is passed to \code{exdqlmDiagnostics()}, the resulting object includes: the estimated sequence $\{\hat u_t\}$, the KL divergence, pplc, the ordered pairs of the QQ-plot, and autocorrelations by lag. By default, the function also produces the QQ-plot, ACF plot and MAP standard forecast error plot.  Examples of the utility of \code{exdqlmDiagnostics()} can be found in the following Section \ref{sec:exs}.

\subsection{k-step-ahead forecast}

For an arbitrary time $\tilde{t}$, the $k$-step-ahead future marginal distribution of the quantile is

\begin{equation}
{\bf F}'_{\tilde{t}+k} {\boldsymbol \theta}_{\tilde{t}+k}| y_1,\dots,y_{\tilde{t}} \sim \text{N}({\bf F}'_{\tilde{t}+k} a_{\tilde{t}}(k),{\bf F}'_{\tilde{t}+k}{\bf R}_{\tilde{t}}(k){\bf F}_{\tilde{t}+k})
\end{equation}

where ${\bf a}_{\tilde{t}}(k) = {\bf G}_{\tilde{t}+k} {\bf a}_{\tilde{t}}(k-1)$, ${\bf R}_{\tilde{t}}(k) = {\bf G}_{\tilde{t}+k} {\bf R}_{\tilde{t}}(k-1) {\bf G}_{\tilde{t}+k}' + {\bf W}_{\tilde{t}+k}$, ${\bf a}_{\tilde{t}}(0) = {\bf m}_{\tilde{t}}$, and ${\bf R}_{\tilde{t}}(0) = {\bf C}_{\tilde{t}}$, with ${\bf m}_{\tilde{t}}$ and ${\bf C}_{\tilde{t}}$ denoting the filtered mean and covariance of $\boldsymbol{\theta}_{\tilde{t}}$, respectively. This distribution is implemented in the function \code{exdqlmForecast} which is compatible with the outputs from both the MCMC and LDVB algorithms. Table \ref{table:fore} shows the input parameters used in the function, as well as their mathematical symbols seen in this section. The output of \code{exdqlmForecast()} includes the parameters of the forecasted distribution as well as ${\bf a}_{\tilde{t}}(k)$ and ${\bf R}_{\tilde{t}}(k)$ for the $k$ steps.

\subsection{Lake Huron}

\subsection{Sunspots}
\label{sec:ex2}

For our next example, we will use the yearly Sunspot time series from the package \pkg{datasets}. The data are yearly counts for sunspots from 1700 to 1988, shown in Figure \ref{fig:ex2quant}. With this example we will show how to: use the \pkg{dlm} package to create the state-space model, combine blocks of a state-space model, apply the LDVB algorithm, perform visual model diagnostics to compare the exDQLM with the DQLM, and use those diagnostics for model selection.

The solar cycle impacts astronauts in space as well as lives and technology on earth. Of particular interest to scientists is the solar max, or period in which the sun is most active \cite{sunspot}, therefore we will explore the $0.85$ quantile in this example. Again, we begin by building the state-space structure used to estimate the quantile. Here we choose to model the quantile with a first order polynomial trend component as well as a Fourier form seasonal component that incorporates a fundamental period every 11 observations (years), as well as some of its corresponding harmonics. This results in a dynamic model with a total of 9 state parameters, 1 for the first order polynomial component and 8 for the seasonal component. The \pkg{exdqlm} functions are compatible with time-invariant \code{dlm} objects from the \pkg{dlm} package. For example, we create the first order trend component with the function \code{dlmModPoly()} from the \pkg{dlm} package, then use \code{as.exdqlm()} to turn the output into an \code{exdqlm} object.
\begin{CodeChunk}

\end{CodeChunk}

\subsection{Big Tree water flows}
\label{sec:ex3}

\begin{figure}[t]
\centering
\includegraphics[width=1\textwidth]{ex3data.png}
\caption{Example 3: Big Tree water flow. Top: Average monthly water flow at the Big Tree gauge of the San Lorenzo river in Santa Cruz County, CA, plotted on the log-scale. Bottom: Monthly values of Ni\~no 3.4. Both time series span January 1937 through April 2021. }
\label{fig:ex3data}
\end{figure}

For our final example, we consider the average monthly natural water flow (cubic feet per second) at the Big Tree gauge of the San Lorenzo river in Santa Cruz County, CA from January 1937 through April 2021 \citep{btflow}, for a total of 1012 observations. The dataset is included in our package as \code{BTflow}. With this example we show how to build a state-space structure manually (particularly for a dynamic quantile regression), apply our transfer function exDQLM, examine individual components of the estimated quantiles, and perform quantitative model diagnostics to compare models.

For this particular dataset, the interest is in low quantiles as a possible indicator of drought, thus we will consider the $p_0=0.15$ quantile. To investigate any climatological influence on this low quantile, we include the Ni\~no 3.4 index \citep{nino34} as in input of our exDQLM. The Ni\~no 3.4 index is a measure of how anomalously warm or cool the average sea surface temperature is in the Ni\~no 3.4 region ($5^\circ$S-$5^\circ$N, $170^\circ$W-$120^\circ$W) of the Pacific Ocean. It is one of the most commonly used indices to define El Ni\~no and La Ni\~na events. The monthly Ni\~no 3.4 dataset used in our analysis is also included in the package as \code{nino34}. We model the relationship between Ni\~no 3.4 and the quantile with two model structures: a dynamic quantile regression and a transfer function as seen in Section \ref{sec:tf}. To avoid modeling problems caused by the scale and extremes in the Big Tree dataset, we fit the following models to the time series on the log scale seen in Figure \ref{fig:ex3data}.

In addition to the components that capture the relationship with the Ni\~no 3.4 index, we also include a first order polynomial trend component as well as a Fourier form seasonal component that includes a single periodic component with period 12. The Big Tree water flow time series follows a period of twelve months, of which we include only the first (annual) harmonic.
\begin{CodeChunk}
\begin{CodeInput}
R> trend.comp = polytrendMod(1, m0 = 3, C0 = 0.1)
R> seas.comp = seasMod(p = 12, h = 1, C0 = diag(1,2))
R> model = trend.comp + seas.comp
\end{CodeInput}
\end{CodeChunk}

Next, we create the dynamic regression component for our first model using \code{regMod()}. Combining this with the trend and seasonal component we have our complete state-space structure for the dynamic quantile regression model.
\begin{CodeChunk}
\begin{CodeInput}
R> reg.comp = regMod(nino34, m0 = 1, C0 = 1)
R> model.w.reg = model + reg.comp
\end{CodeInput}
\end{CodeChunk}

To allow the Ni\~no 3.4 input components of both models to vary over time, we set the respective discount factors to be $0.95$ (this is done via the parameter \code{df} in \code{exdqlmLDVB()} and the parameter \code{tf.df} in \code{transfn_exdqlmLDVB()}). Alternatively, we set the discount factors to 1 and 0.9 for the trend and seasonal components, respectively. This allows the seasonal amplitudes to evolve over time, while any further variation in the $0.15$ quantile from year to year will solely be attributed to the effects of the Ni\~no 3.4 input in our models.

The transfer function model is created within the function \code{transfn_exdqlmLDVB()} and follows the structure detailed in Section \ref{sec:tf}. Optimization using the KL divergence suggests the effects of the Ni\~no 3.4 index decay at an optimal transfer function rate of $\lambda = 0.85$.
\begin{CodeChunk}
\begin{CodeInput}
R> possible.lams = seq(0.1,0.9,0.05)
R> KLs <- vector("numeric")
R> ref.samp = rnorm(length(BTflow))
R> for(i in 1:length(possible.lams)){
R>   temp.M2 = transfn_exdqlmLDVB(y = log(BTflow), p0 = 0.15, model = model,
+                               df = c(1,0.9), dim.df = c(1,2),
+                               X = nino34, tf.df = c(0.95), lam = possible.lams[i],
+                               tf.m0 = c(0,0), tf.C0 = diag(c(0.1,0.005),2),
+                               sig.init = 0.1, gam.init = - 0.1,
+                               tol = 0.05, verbose = FALSE)
R>   temp.check = exdqlmDiagnostics(temp.M2, plot = FALSE,
+                            ref = ref.samp)
R>   KLs = c(KLs,temp.check$m1.KL)
R> }
R> # optimal lambda based off KL divergence
R> possible.lams[which.min(KLs)]
\end{CodeInput}
\begin{CodeOutput}
[1] 0.85
\end{CodeOutput}
\end{CodeChunk}

\subsection{Static exAL regression on a simulated heteroskedastic example}
\label{sec:ex4static}

To illustrate the static routines under sparse shrinkage, we consider a correlated Gaussian regression benchmark for which the target conditional quantile is known exactly. Let
\[
{\bf x}_i \sim N_8({\bf 0}, \Sigma_x),\qquad (\Sigma_x)_{jk} = 0.5^{|j-k|},
\]
with sparse signal
\[
\boldsymbol{\beta}^\star = (3,\,1.5,\,0,\,0,\,2,\,0,\,0,\,0)^\top.
\]
For each target quantile level \(p_0\), we generate
\[
Y_i^{(p_0)} = {\bf x}_i^\top \boldsymbol{\beta}^\star + \sigma_\varepsilon \{Z_i - \Phi^{-1}(p_0)\},
\qquad Z_i \sim N(0,1),\qquad \sigma_\varepsilon = 1.5.
\]
Then
\[
Q_Y(p_0 \mid {\bf x}_i) = {\bf x}_i^\top \boldsymbol{\beta}^\star,
\]
so the true \(p_0\)-quantile is the same sparse linear signal at each fitted quantile level. We use \(n = 160\) training observations and an independent holdout sample of size \(800\), and fit separate static exAL models at \(p_0 \in \{0.05,0.25,0.50\}\). Both fits use the regularized horseshoe prior of \cite{nishimura2023shrunken} via \code{beta_prior = "rhs_ns"} with \(\tau_0 = 0.15\), fixed slab \(\zeta^2 = 9\), and an unshrunk intercept. We fit \code{exal_static_LDVB()} and \code{exal_static_mcmc()}; the MCMC routine uses its default \code{slice} kernel and is warm-started from the LDVB fit. The lower-tail LDVB fit at \(p_0 = 0.05\) converges more slowly, so we allow a larger LDVB iteration budget there. Setting \code{dqlm.ind = TRUE} would recover the AL special case discussed in Section \ref{sec:exdqlm}, but here we focus on the general static exAL model with sparse shrinkage.

\begin{CodeChunk}
\begin{CodeInput}
R> Sigma.x = 0.5 ^ as.matrix(dist(1:8))
R> beta.true = c(3, 1.5, 0, 0, 2, 0, 0, 0)
R> rhs.ctrl = list(tau0 = 0.15, a_zeta = 2, b_zeta = 9,
                   zeta2_fixed = 9, shrink_intercept = FALSE)
R> sim.y = function(X.raw, p0) {
      drop(X.raw %*% beta.true) + 1.5 * (rnorm(nrow(X.raw)) - qnorm(p0))
    }
R> X.raw = MASS::mvrnorm(160, mu = rep(0, 8), Sigma = Sigma.x)
R> X = cbind(1, X.raw)
R> p.grid = c(0.05, 0.25, 0.50)
R> y.train = lapply(p.grid, function(p0) sim.y(X.raw, p0))
R> M.ldvb = lapply(p.grid, function(p0) exal_static_LDVB(
+                    y = y.train[[which(p.grid == p0)]], X = X, p0 = p0,
+                    beta_prior = "rhs_ns",
+                    beta_prior_controls = rhs.ctrl,
+                    max_iter = ifelse(p0 == 0.05, 420, 260),
+                    tol = 1e-4, verbose = FALSE))
R> M.mcmc = lapply(p.grid, function(p0) exal_static_mcmc(
+                    y = y.train[[which(p.grid == p0)]], X = X, p0 = p0,
+                    beta_prior = "rhs_ns",
+                    beta_prior_controls = rhs.ctrl,
+                    n.burn = 400, n.mcmc = 300,
+                    init.from.vb = TRUE,
+                    verbose = FALSE))
\end{CodeInput}
\end{CodeChunk}

For static fits, the helper function \code{exalDiagnostics()} summarizes fitted quantiles on a common design matrix and, when a reference quantile is supplied, reports truth-recovery metrics directly. For example, on an independent holdout design:

\begin{CodeChunk}
\begin{CodeInput}

R> X.hold = cbind(1, X.hold.raw)
R> ref.hold = drop(X.hold %*% c(0, beta.true))
R> y.hold = sim.y(X.hold.raw, 0.25)
R> exalDiagnostics(M.ldvb[[2]], M.mcmc[[2]],
+                  X = X.hold, y = y.hold, ref = ref.hold,
+                  plot = FALSE)
\end{CodeInput}
\end{CodeChunk}

Black diamonds denote the true non-intercept coefficients, orange intervals denote LDVB 95\% posterior intervals, and blue intervals denote MCMC 95\% posterior intervals. Panels correspond to p0 = 0.05, 0.25, and 0.50.}

Figure \ref{fig:ex4static} shows that both methods recover the active coefficients and shrink the null coefficients toward zero across the lower tail, lower-middle quantile, and median. The lower-tail fit at \(p_0 = 0.05\) is the most demanding case, yet both methods recover the sparse signal cleanly; on this run, LDVB is slightly better on the active-signal RMSE and holdout quantile RMSE while still requiring the larger iteration budget. At \(p_0 = 0.25\), the two fits remain qualitatively aligned, with MCMC modestly closer to the true sparse signal. At \(p_0 = 0.50\), both fits again track the true sparse pattern closely, with LDVB slightly sharper on the active coefficients and holdout quantile recovery while MCMC shrinks the null coefficients more aggressively.

Table \ref{tab:ex4static} reinforces the visual comparison. The two methods are competitive across all three quantile levels, but the relative advantage is not uniform: LDVB is slightly stronger in the lower-tail and median runs on the active-signal and holdout-recovery metrics, whereas MCMC is stronger at \(p_0 = 0.25\) and also tends to shrink the null coefficients more aggressively. In runtime terms, the two methods are of roughly comparable cost at the hardest lower-tail fit, while LDVB is clearly faster away from that case: it is about 1.9 times faster at \(p_0 = 0.25\) and about 7.9 times faster at \(p_0 = 0.50\). The main message is that static LDVB can deliver substantial computational savings as a first-pass approximation, but this speed comes with the usual variational tradeoff that posterior uncertainty quantification may be less faithful than the exact MCMC baseline. Accordingly, the static \code{rhs_ns} workflow supports both roles cleanly: LDVB for rapid screening and model exploration, and static MCMC as the reference method when higher-fidelity posterior uncertainty quantification is required.

\section{Conclusion}
\label{sec:summary}

Our exDQLM framework provides a robust and versatile tool for dynamic quantile modeling, implemented in the presented \pkg{exdqlm} package. Relevant features of the data can easily be incorporated in the evolution of the quantile. The MCMC algorithm provides a method for exact posterior inference. Alternatively, the LDVB algorithm provides a method for fast approximate inference enabling estimation for large data sets, quick examination of relevant model features, and model selection. Further, our transfer function model provides a straightforward method for quantifying a nonlinear relationship between a quantile and response. Finally, our forecast and diagnostic routines facilitate further model exploration. These capabilities have been discussed in detail and demonstrated with a set of examples. The \pkg{exdqlm} package makes these methods, and more generally, dynamic quantile modeling, accessible to all levels of users.
