\title{\pkg{exdqlm}: An \proglang{R} Package for Estimation and Analysis of Flexible Dynamic Quantile Linear Models}

\Plainauthor{Antonio Aguirre, Raquel Barata, Raquel Prado, Bruno Sans\'{o}}

\Shorttitle{\pkg{exdqlm}: Flexible Dynamic Quantile Linear Models} %% a short title (if necessary)


\Abstract{ We present the \proglang{R} package \pkg{exdqlm} for Bayesian quantile regression, with primary emphasis on dynamic state-space quantile models for time series. The package is built around flexible dynamic quantile linear models (exDQLMs), which use the extended asymmetric Laplace (exAL) family, a parametric extension of the asymmetric Laplace (AL) commonly used in quantile regression. The software provides exact posterior inference via Markov chain Monte Carlo (MCMC) and fast approximate posterior inference via variational Bayes (VB), supporting posterior uncertainty quantification while also providing a computationally efficient option for longer time series. It also supports static exAL quantile regression with regularized priors, transfer-function models for nonlinear input effects at a given quantile, post hoc posterior-predictive synthesis across separately fitted quantiles, forecasting, and several quantitative and visual diagnostics for model evaluation. }

\Keywords{dynamic quantile regression, dynamic models, asymmetric Laplace, variational Bayes, coordinate ascent variational inference, quantile regression}

\section{Introduction}

\aA{Quantile regression has become a widely used alternative to mean-based modeling in static settings, but dynamic quantile methods and accompanying software remain comparatively limited. In Bayesian parametric quantile regression, the asymmetric Laplace (AL) likelihood has played a central role because it is linked to the check-loss function used in classical quantile regression \citep{yu2001bayesian,koenkerquantile} and admits convenient mixture representations for posterior computation. These computational advantages have made AL-based models attractive, but the AL can be restrictive as an error model. More flexible error distributions have been studied extensively in the Bayesian nonparametric literature \citep{kottas2009bayesian,reich2009flexible,taddy2010bayesian}; however, the resulting methods can be computationally demanding for dynamic models and longer time series. \cite{yan2025family} introduce the extended asymmetric Laplace (exAL), a parametric extension of the AL that preserves its computational advantages while relaxing some of its restrictive features. \cite{me} use this family to develop a flexible dynamic quantile linear model (exDQLM), which is the model implemented in the \proglang{R} package \pkg{exdqlm}.}

\aA{The package \pkg{exdqlm} provides a unified workflow for Bayesian inference, forecasting, and model assessment for quantile models fitted one quantile at a time. Its core functionality is dynamic state-space quantile modeling, supported by exact posterior simulation via MCMC and a faster variational Bayes (VB) approximation. The package also includes transfer-function exDQLMs for nonlinear input effects at a given quantile, static exAL quantile regression with shrinkage priors, and a synthesis step that combines anchor-specific posterior predictive draws from separately fitted quantile models into a unified posterior predictive distribution. Forecasting and a range of visual and quantitative diagnostics are available throughout. The package is released under the MIT License and is available from the Comprehensive R Archive Network (CRAN) at \url{https://CRAN.R-project.org/package=exdqlm}.}

\aA{The current \proglang{R} software ecosystem for quantile regression is broad, but the capabilities most relevant to this paper are distributed across software with different aims. The frequentist package \pkg{quantreg} \citep{quantreg} remains the broad general-purpose baseline, covering linear, nonlinear, nonparametric, censored, and dynamic quantile regression, together with bootstrap-based and related uncertainty summaries. Its \code{dynrq()} function supports dynamic linear quantile regression through time-series regression formulas involving lags, differences, trends, seasonal factors, and harmonic terms, while preserving time-series attributes. However, the dynamic structure enters through the design matrix rather than through latent states: once these regressors are constructed, estimation proceeds through the standard \pkg{quantreg} fitting routines for a fixed coefficient vector. In that sense, \code{dynrq()} is a regression-style time-series interface rather than a latent-state state-space model, so it does not provide filtered or smoothed state summaries.}

\aA{Most other \proglang{R} packages relevant here are static, or else they target a different multi-quantile objective. Among the static Bayesian options, \pkg{bayesQR} \citep{bayesQR} provides Bayesian quantile regression under the AL working likelihood with an adaptive-lasso option, whereas \pkg{pqrBayes} \citep{pqrBayes} emphasizes penalized Bayesian quantile regression with spike-and-slab and horseshoe-family shrinkage priors. The broader Bayesian modeling package \pkg{brms} \citep{brmsfamily} is also relevant because it supports AL-based quantile regression within a flexible multilevel, nonlinear, and additive regression framework, and can incorporate some autocorrelation structures. However, it is not organized as a dedicated quantile state-space workflow, and its documentation does not present filtered or smoothed state summaries of the type targeted by \pkg{exdqlm}. Packages such as \pkg{qrjoint} \citep{qrjoint}, \pkg{quantregGrowth} \citep{quantregGrowth}, and \pkg{qrcm} \citep{qrcm} instead address a different cross-quantile objective through joint estimation, non-crossing constraints, or coefficient modeling across quantiles. Other packages focus on specialized static settings, including additive or semiparametric modeling in \pkg{qgam} \citep{qgam}, penalized frequentist fitting across quantiles in \pkg{rqPen} \citep{rqPen}, convolution-smoothed linear quantile regression with efficient gradient-based fitting in \pkg{conquer} \citep{conquer}, and robust or censored-response modeling in \pkg{lqr} \citep{lqr}.}

\aA{Related implementations also exist outside \proglang{R}. In Python, \pkg{statsmodels} \citep{statsmodelsQuantReg} provides linear quantile regression via iterative reweighted least squares. Julia offers \pkg{QuantReg.jl} \citep{QuantRegjl}, and MATLAB includes tools such as \code{fitrqlinear} and \code{fitrqnet} \citep{matlabQuantile}. These implementations are useful reference points for the broader software ecosystem, but they are better viewed as general or static quantile-regression tools than as direct counterparts to a dedicated package for Bayesian dynamic quantile state-space modeling.}

\aA{Viewed against this software ecosystem, \pkg{exdqlm} is not intended to replace broad general-purpose quantile-regression toolkits, additive quantile-regression packages, or software designed specifically for joint multi-quantile estimation. Rather, it addresses a more specific need: a unified Bayesian workflow for dynamic/state-space quantile modeling that combines exact posterior simulation with a fast variational approximation, while also using the more flexible exAL likelihood, supporting static exAL regression with shrinkage priors ranging from ridge to regularized horseshoe variants, and providing transfer-function models, forecasting, diagnostics, and post hoc posterior-predictive synthesis from separately fitted quantile models. General state-space packages such as \pkg{dlm} \citep{dlm} and \pkg{dynr} \citep{dynr} remain useful for constructing dynamic components, but they do not provide dedicated quantile-regression workflows.}

\aA{The remainder of the paper is organized as follows. In Section \ref{sec:exdqlm}, we describe the modeling framework, including the algorithms used for posterior inference and our methods for including nonlinear relationships. We also discuss static quantile regression as a special case of the framework. In Section \ref{sec:diags} we detail the model diagnostics and forecasting method implemented in the package. Then, in Section \ref{sec:exs}, we walk through four step-by-step examples on how to use the package. Finally, in Section \ref{sec:summary}, we present a summary of the package capabilities.}

\section{Extended dynamic quantile linear models}
\label{sec:exdqlm}

Consider a set of scalar time-evolving responses, $y_t$, for times $t=1,\dots,T$. For each $t$, an extended dynamic quantile linear model (exDQLM) for inference on a single $p_0$-th quantile is defined by
\begin{align}
\label{EQ:exDLM}
& \text{Observation equation:} & y_t &= {\bf F}'_t \boldsymbol{\theta}_t + \epsilon_t,  & \epsilon_t & \sim \text{exAL}_{p_0}(0,\sigma,\gamma) \\
\label{EQ:exDLMevo}
& \text{System equation:} & \boldsymbol{\theta}_t &= {\bf G}_t \boldsymbol{\theta}_{t-1} + \boldsymbol{\omega}_t, & \boldsymbol{\omega}_t  & \sim \text{N}({\bf 0},{\bf W}_t).
\end{align}
Here ${\bf F}_t$ is the $q \times 1$ regression vector of the covariates corresponding to the $q \times 1$ parameter vector $\boldsymbol{\theta}_t$ at time $t$, ${\bf G}_t$ is the $q\times q$-dimensional evolution matrix defining the structure of the parameter vector evolution in time, and the normal distribution according to which the system error $\boldsymbol{\omega}_t$ evolves is $q$-variate. The observational errors $\epsilon_t$ of the exDQLM follow an exAL distribution for fixed quantile $p_0$, whose density is denoted as $\text{exAL}_{p_0}(\cdot)$, and it is such that $\int_{-\infty}^0 \text{exAL}_{p_0}(\epsilon_t |0,\sigma,\gamma) \mathrm{d}\epsilon_t = p_0$. Thus, the observation equation in Equation (\ref{EQ:exDLM}) implies that ${\bf F}'_t \boldsymbol{\theta}_t$ corresponds to the $p_0$-quantile of $y_t$,
even if this is not made explicit in the notation above. Table \ref{table:priors} shows the function parameters that define the structure of the dynamic quantile, specifically ${\bf F}_t$ and ${\bf G}_t$ for $t=1,\ldots,T$.

The exAL distribution, introduced by \cite{yan2025family}, extends the location-scale mixture representation of the AL \citep{kozumi2011gibbs}. For fixed $p_0$, the skewness parameter $\gamma$ is restricted to an interval $(L,U)$, where $L$ and $U$ are the negative and positive roots of $g(\gamma) = 1-p_0$ and $g(\gamma) = p_0$, respectively, with $g(\gamma) = 2\Phi(-|\gamma|)\exp(\gamma^2/2)$. \aA{The package also exports direct distributional utilities for the exAL family.} These include \code{dexal()}, \code{pexal()}, \code{qexal()}, and \code{rexal()}, together with \code{get_gamma_bounds()} for recovering this admissible interval. This results in a mixture representation of $\text{exAL}_{p_0}(0,\sigma,\gamma)$ as follows:
\begin{equation}
\label{EQ:exALmix}
\int\int_{\mathbb{R}^+ \times \mathbb{R}^+} \text{N}(y_t |  C(p,\gamma) \sigma |\gamma| s_t + A(p) v_t, \sigma B(p) v_t) \text{Exp}(v_t | \sigma) \text{N}^+(s_t| 0,1) \mathrm{d}v_t \mathrm{d}s_t.
\end{equation}
Here, $p = p(p_0,\gamma) = I(\gamma <0) + \{[p_0 - I(\gamma <0)]/g(\gamma) \}$ where $g(\gamma) = 2\Phi(-|\gamma|)\text{exp}(\gamma^2/2)$, and $\Phi(\cdot)$ denotes the standard normal CDF. $A(p)$, $B(p)$, $C(p,\gamma)$ are the functions of $p$ and $\gamma$: $A(p) = \frac{1-2p}{p(1-p)}$, $B(p) = \frac{2}{p(1-p)}$, $C(p,\gamma) = [I(\gamma>0) -p]^{-1}$. Lastly, $\text{Exp}(v | \sigma)$ denotes the exponential distribution with mean $\sigma$, and $\text{N}^+(s_t| 0,1)$ denotes a normal distribution truncated to the positive reals with mean $0$ and variance $1$. This mixture representation of the exAL can be exploited to rewrite the exDQLM as the following hierarchical model for $t=1,\dots,T$:
\begin{align}
\label{EQ:exDQLMobs}
y_t | \boldsymbol{\theta}_t,\sigma,\gamma,v_t,s_t  & \sim \text{N}(y_t | {\bf F}_t' \boldsymbol{\theta}_t + C(p,\gamma) \sigma |\gamma| s_t + A(p) v_t,\sigma B(p) v_t) \\
v_t, s_t | \sigma & \sim \text{Exp}(v_t | \sigma) N^+(s_t |0,1) \\
\label{EQ:exDQLMevo}
\boldsymbol{\theta}_t | \boldsymbol{\theta}_{t-1},{\bf W}_t & \sim  \text{N}({\bf G}_t \boldsymbol{\theta}_{t-1},{\bf W}_t).
\end{align}
This hierarchical form is leveraged for the posterior inference implemented within the \pkg{exdqlm} package.

Under a Bayesian formulation and given $p_0$, we specify priors for the initial state vector $\boldsymbol{\theta}_0$, scale parameter $\sigma$ and skewness parameter $\gamma$. For $\boldsymbol{\theta}_0$, we assume a $q$-variate conjugate normal prior $\boldsymbol{\theta}_0 \sim \text{N}({\bf m}_0,{\bf C}_0)$. For $\sigma$, we assume a conjugate inverse-gamma prior denoted as $\text{IG}(a_{\sigma},b_{\sigma})$. In many cases a strong prior on $\sigma$ is necessary to guarantee fast posterior convergence and estimation. This is illustrated in the examples of Section \ref{sec:exs}. The parameter $\gamma$ has bounded support over the interval $(L,U)$ where $L$ is the negative root of $g(\gamma) = 1-p_0$ and $U$ is the positive root of $g(\gamma) = p_0$ \citep{yan2025family}. In contrast to the flat prior suggested by \cite{yan2025family}, we find that using a proper prior on the skewness parameter promotes reliable posterior inference. Thus, we place a Student-t distribution truncated to the interval $(L,U)$ as the prior for $\gamma$, i.e. $\gamma \sim \text{t}_{(L,U)}(m_\gamma,s_\gamma)$ with $\nu_\gamma$ degrees of freedom. The prior parameters used in the \pkg{exdqlm} functions, their corresponding mathematical symbols, and default values are shown in Table \ref{table:priors}.

\subsection{Posterior estimation}

The hierarchical representation of the exDQLM facilitates posterior simulation using Markov Chain Monte Carlo (MCMC) algorithms. This is implemented in the function \code{exdqlmMCMC()}. Gibbs sampling steps are used to update the sets of latent variables $s_t$ and $v_t$ for $t=1,\dots,T$. The dynamic regression coefficients are sampled using a forward filtering backwards sampling algorithm \citep{carter1994gibbs,fruhwirth1994data}. \aA{The scale and skewness parameters are updated separately from the latent-state blocks. By default, the package uses a slice-sampling update for the $(\sigma,\gamma)$ block, while joint random-walk Metropolis-Hastings (MH) alternatives are also available. When desired, the chain can be warm-started from the variational approximation; this initialization is separate from the subsequent MCMC proposal kernel.} An example of this functionality can be found in Section \ref{sec:ex1}. We perform a total of $N_{BURN} + N_{MCMC}$ MCMC iterations, discarding the first $N_{BURN}$ and retaining the remaining $N_{MCMC}$ posterior samples. The object created by \code{exdqlmMCMC()} includes posterior samples of all parameters ($\boldsymbol{\theta}_{1:T},v_{1:T},s_{1:T},\gamma,\sigma$), filtered and smoothed summaries for the state vector $\boldsymbol{\theta}_{1:T}$, and diagnostics for the proposal kernel used in the $(\sigma,\gamma)$ update.

For purposes of model selection or large time series data sets, posterior sampling with the MCMC algorithm can be computationally demanding. \aA{To address this, the package includes a variational Bayes (VB) routine for fast approximate posterior estimation. The default routine, \code{exdqlmLDVB()}, uses a mean-field approximation together with a Laplace-Delta treatment of the non-conjugate $(\sigma,\gamma)$ block.} The variational Bayes (VB) framework approximates the posterior distribution with partitioned variational distributions \citep{ostwald2014tutorial,beal2003variational}. Under a mean-field factorization, this induces a blockwise coordinate-ascent variational inference scheme. For the exDQLM, the blocks associated with the dynamic coefficients and latent variables admit recognizable closed-form updates, while the joint variational distribution of $\sigma$ and $\gamma$ is non-conjugate. \aA{Following the non-conjugate VB strategy of \cite{wang2013nonconjugatevb}, we approximate this block on transformed coordinates and use a second-order Delta method to evaluate the expectations of functions of $(\sigma,\gamma)$ needed by the remaining variational updates.} The dynamic regression coefficients are updated within each iteration using a forward filtering backwards smoothing algorithm. Convergence is assessed jointly through the state, scale, skewness, and ELBO diagnostics. After the algorithm reaches convergence with tolerance $\epsilon_{tol}$, $N_{SAMP}$ samples are drawn from the variational distributions, resulting in an approximate sample of the posterior distribution. The output of \code{exdqlmLDVB()} includes the approximate posterior samples as well as the parameters of the variational distributions for all parameters ($\boldsymbol{\theta}_{1:T},v_{1:T},s_{1:T},\gamma,\sigma$). For the $(\gamma,\sigma)$ block, the return also contains the transformed mode, approximate covariance matrix, derived moments, and ELBO terms associated with the Laplace-Delta approximation. \aA{Selected state-space computations in the dynamic routines are implemented in \proglang{C++}, with corresponding \proglang{R} code retained for parity checks and fallback use.}

\subsection{Specification of the evolution covariance via discount factors}
\label{sec:dfs}

Both estimation algorithms, \code{exdqlmMCMC()} and \code{exdqlmLDVB()}, utilize discount factors to specify the time-evolving covariance matrices ${\bf W}_t$ introduced in Equation \eqref{EQ:exDLMevo} \citep{west1985dynamic}. More precisely, we define the evolution variance matrix ${\bf W}_t = \frac{1-\delta}{\delta} {\bf G}_t {\bf C}_{t-1} {\bf G}_t'$ where ${\bf C}_{t-1}$ denotes the posterior variance of the state vector ${\boldsymbol\theta_{t-1}}$ at time $t-1$ and $\delta$ is a discount factor such that $0 < \delta \leq 1$. The discount factor $\delta$ can be specified by the user via function parameter \code{df}. \aA{Selection of discount factors is typically done by optimizing a model-checking criterion. In practice we recommend predictive criteria such as the CRPS or pplc discussed in Section \ref{sec:diags}, with the KL divergence retained as a complementary calibration diagnostic.} A brief example of this method for discount factor selection can be found in Section \ref{sec:ex2}.

Discounting by component of the state-space model is also a functionality of our estimation algorithms. Suppose the state vector is comprised of $h$ components $\boldsymbol{\theta}_{it}$ of dimension $q_i$ for $i=1,\dots,h$ such that $\boldsymbol{\theta}_{t}' = (\boldsymbol{\theta}_{1t}',\dots,\boldsymbol{\theta}_{ht})'$ and $\sum_{i=1}^{h} q_i = q$. If ${\bf W}_{it}$ denotes the $i^{th}$ component of the evolution covariance matrix such that ${\bf W}_t = \text{blockdiag}\{ {\bf W}_{1t},\dots,{\bf W}_{ht}\}$ and $\delta_i$ denotes a discount factor for the $i^{th}$ component, we can define the evolution variance matrix by component as ${\bf W}_{it} = \frac{1-\delta_i}{\delta_i} {\bf G}_{it} {\bf C}_{i,t-1} {\bf G}_{it}'$. Here ${\bf G}_{it}$ and ${\bf C}_{i,t-1}$ denote the $i^{th}$ components of ${\bf G}_t$ and ${\bf C}_{t-1}$, respectively. The function parameters \code{df} and \code{dim.df} allow users to specify a set of component discount factors $(\delta_1,\dots,\delta_h)$ and their dimensions $(q_1,\dots,q_h)$, respectively, within the state-space model. Examples of this feature can be found in Section \ref{sec:exs}. For a full review of discount factors including discounting by component, see \cite{west1985dynamic}.

\subsection{Transfer function model}
\label{sec:tf}

Quantifying the relationship between an input and a quantile of a response is a non-trivial task, particularly when the relationship is nonlinear. In mean-centric dynamic modeling, transfer functions are a straightforward method to incorporate variables which measure the combined effect of current and past regression effects \citep{west1985dynamic}. \aA{Within \pkg{exdqlm}, this extension is handled by augmenting the dynamic state-space representation conditional on a fixed transfer-function rate \(\lambda\), and the resulting workflow is available through both VB and MCMC fitting routines.}

For $t=1,\dots,T$ and transfer input vector ${\bf x}_t \in \mathbb{R}^k$, a transfer-function exDQLM with exponential decay is defined by
\begin{align}
\label{EQ:begin_transfer}
y_t | \boldsymbol{\theta}_t,\zeta_t,\gamma,\sigma  & \sim \text{exAL}_{p_0}({\bf F}'_t \boldsymbol{\theta}_t +  \zeta_{t},\sigma,\gamma) \\
\boldsymbol{\theta}_t | \boldsymbol{\theta}_{t-1},{\bf W}_t^\theta  & \sim  \text{N}({\bf G}_t \boldsymbol{\theta}_{t-1},{\bf W}_t^\theta) \\
\label{EQ:transfer_effect}
\zeta_{t}|\zeta_{t-1},\boldsymbol{\psi}_{t-1},\omega_{t} & \sim \text{N}(\lambda \zeta_{t-1} + {\bf x}'_{t}\boldsymbol{\psi}_{t-1},\omega_{t}) \\
\label{EQ:end_transfer}
\boldsymbol{\psi}_{t}|\boldsymbol{\psi}_{t-1},{\bf W}_{t}^{\psi} & \sim \text{N}(\boldsymbol{\psi}_{t-1},{\bf W}_{t}^{\psi}).
\end{align}
Here $\zeta_t \in \mathbb{R}$ denotes the accumulated transfer effect on the quantile at time $t$, while $\boldsymbol{\psi}_t \in \mathbb{R}^k$ is the vector of instantaneous transfer coefficients associated with the components of ${\bf x}_t$. The mean contribution of the transfer block is therefore $\lambda \zeta_{t-1} + {\bf x}'_t \boldsymbol{\psi}_{t-1}$: the inner product ${\bf x}'_t \boldsymbol{\psi}_{t-1}$ captures the contemporaneous effect of the current inputs, and the parameter $\lambda\in[0,1]$ controls how much of the previous accumulated effect is propagated forward. For $0<\lambda<1$, this propagated contribution decays geometrically. More explicitly, the contribution of ${\bf x}_t$ to the quantile at time $t+h$ is $\lambda^h {\bf x}'_t \boldsymbol{\psi}_{t-1}$. Therefore, for each $t$ and tolerance $\epsilon>0$, any
\[
k_t \geq \frac{\log(\epsilon)-\log\!\left(|{\bf x}'_t \boldsymbol{\psi}_{t-1}|\right)}{\log(\lambda)}
\]
provides a lower bound on the number of time steps until the magnitude of the propagated effect falls below $\epsilon$. The package reports the median of this bound over time for $\epsilon = 1\mathrm{e}{-3}$.

Conditional on a fixed $\lambda$, say $\tilde\lambda$, \aA{we augment the dynamic fitting routines} to incorporate the transfer-function structure by replacing ${\bf F}_t$, $\boldsymbol{\theta}_t$, ${\bf G}_t$, and ${\bf W}_t^\theta$ in Equations \eqref{EQ:exDLM}-\eqref{EQ:exDLMevo} with
\[
\tilde{\bf F}'_t = ({\bf F}'_t, 1, {\bf 0}'_k), \qquad
\tilde{\boldsymbol{\theta}}'_t = (\boldsymbol{\theta}'_t,\zeta_t,\boldsymbol{\psi}'_t),
\]
\[
\tilde{\bf G}_t = \text{blockdiag}\{{\bf G}_t, {\bf G}^{tf}_t\},
\qquad
{\bf G}^{tf}_t =
\begin{pmatrix}
\tilde\lambda & {\bf x}'_t \\
{\bf 0}_k & {\bf I}_k
\end{pmatrix},
\]
and
\[
\tilde{\bf W}_t = \text{blockdiag}\{{\bf W}_t^\theta,\omega_t,{\bf W}_t^\psi\}.
\]
This augmentation is implemented in the functions \code{exdqlmTransferLDVB()} and \code{exdqlmTransferMCMC()}, which extend the routines found in \code{exdqlmLDVB()} and \code{exdqlmMCMC()}.

Similar to the specification of ${\bf W}_t^\theta$ discussed in Section \ref{sec:dfs}, discount factors are used to specify the transfer block variances $\omega_t$ and ${\bf W}_t^\psi$. In the package, \code{tf.df} can be supplied as a single shared discount factor, as a pair $(\delta_\zeta,\delta_\psi)$ with a shared value for the whole $\boldsymbol{\psi}_t$ block, or componentwise across $(\zeta_t,\psi_{1,t},\dots,\psi_{k,t})$. A conjugate normal prior on $(\zeta_{0},\boldsymbol{\psi}'_{0})' \sim N({\bf m}_0^{tf},{\bf C}_0^{tf})$, with ${\bf m}_0^{tf}\in\mathbb{R}^{k+1}$ and ${\bf C}_0^{tf}\in\mathbb{R}^{(k+1)\times(k+1)}$, completes the transfer-function model extension. Table \ref{table:tf} shows the additional parameters used in the transfer-function fitting routines, as well as their mathematical symbols discussed in this section.

\subsection{Special cases, including static quantile regression}
\label{sec:special}

The first special case of our exDQLM arises from the fact that the exAL is an extension of the AL \citep{yan2025family}. When $\gamma = 0$, the
observational error $\epsilon_t$ in Equation \eqref{EQ:exDLM} reduces such that $\epsilon_t \sim \text{exAL}_{p_0}(\epsilon_t |0,\sigma,\gamma) = \text{AL}_{p_0}(\epsilon_t |0,\sigma)$. This special case with observational errors distributed independently from an AL is the dynamic quantile linear model (DQLM) presented in \cite{gonccalves2017dynamic}. The DQLM can be implemented with our package using the parameter \code{dqlm.ind = TRUE}.

Non-time-varying quantile regression is also a special case of our methodology. More specifically, when discount factors are set to 1, the evolution covariance ${\bf W}_t$ reduces to ${\bf 0}$ resulting in non-time-varying parameters \citep{west1985dynamic}. Thus, the model reduces to the static exAL regression methodology of \cite{yan2025family}. \aA{The static exAL special case is implemented directly in the package.} This model can be implemented using our functions \code{exalStaticMCMC()} or \code{exalStaticLDVB()}, which provide MCMC and VB algorithms, respectively.

The final special case is the intersection of the previous two. A static quantile regression model with observational errors distributed according to an AL corresponds to Bayesian linear quantile regression under the AL working likelihood, first presented in \cite{yu2001bayesian}. This methodology, utilized to create the Bayesian package for quantile regression \pkg{bayesQR}, can also be implemented with our package using functions \code{exalStaticMCMC()} or \code{exalStaticLDVB()} with parameter \code{dqlm.ind = TRUE}, which acts as a convenience flag fixing $\gamma = 0$.

Examples of these special cases can be found in Section 4. A static quantile regression example specifically can be found in Section \ref{sec:ex4static}. For further details on the methodology outlined in this section, see \cite{me}.

\section{Model Diagnostics and Forecasting}
\label{sec:diags}

To evaluate the quantile inference and predictive performance of the exDQLM, several quantitative and visual model diagnostics are included in the package.

The one-step-ahead predictive distribution function introduced by \cite{rosenblatt1952remarks} is a useful diagnostic for models in which the marginal forecast distributions are unrecognizable or difficult to compute \citep{huerta2003time,prado2006multivariate}. For the exDQLM, this distribution is defined as
\begin{align}
u_t = \text{Pr}(Y_t \leq y_t |y_{1:t-1},v_{1:T},s_{1:T},\sigma,\gamma), % \Phi(y_t| y_{1:t-1},v_{1:T}},s_{1:T},\sigma,\gamma) =
\end{align}
with $y_{1:t-1} =y_1,\dots,y_{t-1}$, $v_{1:T}= (v_1,\dots,v_T)$ and $s_{1:T}= (s_1,\dots,s_T)$. Here $u_t$ defines an independent sequence which is uniformly distributed on the interval $(0,1)$ \citep{rosenblatt1952remarks}. Conditional on $\{v_{1:T},s_{1:T},\sigma,\gamma\}$, the forecast distribution of $y_t$ within the exDQLM is normally distributed. Therefore, conditional on MAP estimates $\{\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma\}$ we estimate the sequence as
\begin{align}
\hat{u}_t = \Phi(y_t| y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma)
\end{align}
where $\Phi$ denotes the normal CDF.

Using this estimated sequence $\{\hat u_t\}$, we can assess the model performance in several ways. First, we visually examine the correlation of the estimated sequence via an ACF plot. Second, by transforming the values with a standard normal inverse CDF, we compare their distributional shape to that of a standard normal with a normal QQ-plot. To quantify the divergence of this transformed sequence from normality we compute the KL divergence \citep{kullback1951information}, $\text{KL}(h,\phi) = \int_{-\infty}^{\infty} h(x)\log \frac{h(x)}{\phi(x)}\mathrm{d}x$, where $h$ denotes the numerically approximated density of our transformed sample and $\phi$ is the standard normal density. A smaller KL divergence suggests a superior model fit. Lastly, the MAP standardized forecast errors,
\begin{align}
\frac{y_t-\E[y_t | y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma]}{\sqrt{\text{Var}(y_t|y_{1:t-1},\hat{v}_{1:T},\hat{s}_{1:T},\hat\sigma,\hat\gamma)}}
\end{align}
can also be used as an additional graphical aid in examining the distribution at specific time points. These MAP standardized forecast errors are included in the output of the \code{exdqlmMCMC()} and \code{exdqlmLDVB()} functions.

\aA{To connect the predictive criteria used below, let $\rho_p(x)=x[p-I(x<0)]$ denote the check loss at level $p$. In addition to the calibration-based diagnostics above, we compute the mean continuous ranked probability score (CRPS) from the posterior predictive draws \citep{gneiting2007}. For a predictive distribution $G$ with quantile function $G^{-1}$ and an observation $y$, the CRPS is
\[
\text{CRPS}(y,G)=\mathbb{E}_{Y\sim G}|Y-y|-\frac{1}{2}\mathbb{E}_{Y,Y'\sim G}|Y-Y'|
=2\int_0^1 \rho_p\!\left(y-G^{-1}(p)\right)\,dp,
\]
This representation shows that the CRPS aggregates check-loss performance over all quantile levels, so it evaluates the full predictive distribution rather than a single target quantile. It rewards forecasts that are both sharp and well calibrated, and it reduces to the absolute error, and hence to the MAE after averaging, for deterministic forecasts. Smaller values indicate better predictive performance.}

For a final diagnostic, we include \cite{gelfand1998model} posterior predictive loss criterion (pplc), which evaluates the posterior predictive distribution under the target-level check loss $\rho_{p_0}$. That is,
\begin{align}
\text{pplc} = \sum_t \E[\rho_{p_0} (y_t^{obs}-y^{rep}_t)|y_{1:T}],
\end{align}
where the expectation is with respect to the posterior predictive distribution of $y_t$, $p(y^{rep}_t| y_{1:T})$. This distribution is used to generate a replicate dataset and is the expected value of the observation equation \eqref{EQ:exDLM} with respect to the posterior distributions of the parameters. Thus, the CRPS summarizes the full predictive distribution through an integrated check loss, whereas the pplc keeps the target quantile explicit through $\rho_{p_0}$. Smaller pplc values suggest a superior model fit. Samples from the posterior replicate distributions are included in the output of the \code{exdqlmMCMC()} and \code{exdqlmLDVB()} functions.

The function \code{exdqlmDiagnostics()} implements the model checking criteria discussed in this section. When an output from either \code{exdqlmMCMC()} and \code{exdqlmLDVB()} is passed to \code{exdqlmDiagnostics()}, the resulting object includes: the estimated sequence $\{\hat u_t\}$, the KL divergence, the mean CRPS, pplc, the ordered pairs of the QQ-plot, and autocorrelations by lag. By default, the function also produces the QQ-plot, ACF plot and MAP standard forecast error plot.  Examples of the utility of \code{exdqlmDiagnostics()} can be found in the following Section \ref{sec:exs}.

\subsection{k-step-ahead forecast}

For an arbitrary time $\tilde{t}$, the $k$-step-ahead future marginal distribution of the quantile is

\begin{equation}
{\bf F}'_{\tilde{t}+k} {\boldsymbol \theta}_{\tilde{t}+k}| y_1,\dots,y_{\tilde{t}} \sim \text{N}({\bf F}'_{\tilde{t}+k} a_{\tilde{t}}(k),{\bf F}'_{\tilde{t}+k}{\bf R}_{\tilde{t}}(k){\bf F}_{\tilde{t}+k})
\end{equation}

where ${\bf a}_{\tilde{t}}(k) = {\bf G}_{\tilde{t}+k} {\bf a}_{\tilde{t}}(k-1)$, ${\bf R}_{\tilde{t}}(k) = {\bf G}_{\tilde{t}+k} {\bf R}_{\tilde{t}}(k-1) {\bf G}_{\tilde{t}+k}' + {\bf W}_{\tilde{t}+k}$, ${\bf a}_{\tilde{t}}(0) = {\bf m}_{\tilde{t}}$, and ${\bf R}_{\tilde{t}}(0) = {\bf C}_{\tilde{t}}$, with ${\bf m}_{\tilde{t}}$ and ${\bf C}_{\tilde{t}}$ denoting the filtered mean and covariance of $\boldsymbol{\theta}_{\tilde{t}}$, respectively. \aA{This distribution is implemented in the function \code{exdqlmForecast()}, which is compatible with the outputs from both the MCMC and VB algorithms.} Table \ref{table:fore} shows the input parameters used in the function, as well as their mathematical symbols seen in this section. The output of \code{exdqlmForecast()} includes the parameters of the forecasted distribution as well as ${\bf a}_{\tilde{t}}(k)$ and ${\bf R}_{\tilde{t}}(k)$ for the $k$ steps.

R> exdqlmForecast(start.t = length(LakeHuron), k = 8, m1 = M5,
+               fFF = fFF, fGG = fGG, plot = TRUE, add = TRUE,
+               cols = c("forest green","green"))
\end{CodeInput}
\end{CodeChunk}

This generates a plot of the data with the forecasted quantile estimates and 95\% credible intervals (seen after 1972), along with the filtered quantile estimates and 95\% credible intervals (before 1972), for reference. Results can be seen in the top-right panel of Figure \ref{fig:ex1quants}. The percentage in the CrIs can be modified with the parameter \code{cr.percent}.

\aA{When several quantile models are fitted separately, the package also allows their posterior predictive draws to be combined into a single coherent predictive distribution through \code{quantileSynthesis()}. Each fitted model contributes local predictive information near its target quantile, and the synthesis step interpolates across these anchor-specific draws to obtain one posterior predictive distribution. When independently fitted quantiles induce crossing, optional monotonicity corrections based on isotonic adjustment and monotone rearrangement are available \citep{barlow1972,chernozhukov2010}.} For the Lake Huron example, we apply this routine to the posterior predictive samples stored in \code{M5}, \code{M50}, and \code{M95} over the observed period. \aA{Since these posterior predictive draws are produced from the smoothed state draws, the lower panels of Figure \ref{fig:ex1quants} display the synthesized posterior predictive 95\% interval over the observed and forecast windows.} For the eight-step forecast horizon, we first generate future predictive draws from each fitted model using the corresponding forecasted quantile distributions together with posterior draws of $\sigma$ and $\gamma$, and then apply the same synthesis step. This provides a single predictive envelope while preserving the separate quantile-specific fits.

\aA{A direct call for the observed posterior predictive draws is given below:}
\begin{CodeChunk}
\begin{CodeInput}
R> syn.obs = quantileSynthesis(
+               draws_list = list(M5$samp.post.pred, M50$samp.post.pred,
+                                  M95$samp.post.pred),
+               p = c(0.05, 0.50, 0.95),
+               T_expected = length(LakeHuron))
\end{CodeInput}
\end{CodeChunk}

The function \code{exdqlmForecast()} returns the forecast summaries, but not matrices of future observations. To construct those, we first sample future quantile values from the forecast distribution and then map them through \code{rexal()} using matched posterior draws of $\sigma$ and $\gamma$. For example, for the $p_0 = 0.05$ fit:
\begin{CodeChunk}
\begin{CodeInput}
R> q.fore = sweep(matrix(rnorm(8 * n.samp), 8, n.samp), 1,
+                 sqrt(fc05$fQ), "*") + fc05$ff
R> future.M5$ydraw = vapply(1:n.samp, function(j)
+                 rexal(8, p0 = M5$p0, mu = q.fore[, j],
+                       sigma = sigma.draws[j], gamma = gamma.draws[j]),
+                 numeric(8))
\end{CodeInput}
\end{CodeChunk}

\aA{Here \code{sigma.draws} and \code{gamma.draws} denote matched retained posterior draws from the fitted model. Analogous matrices are generated for \code{M50} and \code{M95}.}

We then collect these matrices into \code{future.draws} and synthesize them in the same way:
\begin{CodeChunk}
\begin{CodeInput}
R> future.draws = list(future.M5$ydraw, future.M50$ydraw,
+                      future.M95$ydraw)
\end{CodeInput}
\end{CodeChunk}

\begin{CodeChunk}
\begin{CodeInput}
R> syn.fore = quantileSynthesis(
+               draws_list = future.draws,
+               p = c(0.05, 0.50, 0.95),
+               T_expected = 8)
\end{CodeInput}
\end{CodeChunk}

\subsection{Sunspots}
\label{sec:ex2}

For our next example, we use the yearly Sunspot time series from the package \pkg{datasets}. The data are yearly counts for sunspots from 1700 to 1988, shown in Figure \ref{fig:ex2quant}. \aA{In this example we show how to use the \pkg{dlm} package to create the state-space model, combine blocks of a state-space model, apply the VB approximation through \code{exdqlmLDVB()}, perform visual model diagnostics to compare the exDQLM with the DQLM, and use those diagnostics for discount-factor selection.}

The solar cycle impacts astronauts in space as well as lives and technology on earth. Of particular interest to scientists is the solar max, or period in which the sun is most active \cite{sunspot}, therefore we will explore the $0.85$ quantile in this example. Again, we begin by building the state-space structure used to estimate the quantile. Here we choose to model the quantile with a first order polynomial trend component as well as a Fourier form seasonal component that incorporates a fundamental period every 11 observations (years), as well as some of its corresponding harmonics. This results in a dynamic model with a total of 9 state parameters, 1 for the first order polynomial component and 8 for the seasonal component. The \pkg{exdqlm} functions are compatible with time-invariant \code{dlm} objects from the \pkg{dlm} package. For example, we create the first order trend component with the function \code{dlmModPoly()} from the \pkg{dlm} package, then use \code{as.exdqlm()} to turn the output into an \code{exdqlm} object.

\end{CodeChunk}

To combine the two components into a single state-space structure, we add the two \code{exdqlm} objects.

\subsection{Big Tree water flows}
\label{sec:ex3}

\begin{figure}[t]
\centering
\includegraphics[width=1\textwidth]{ex3data.png}
\caption{Example 3: Big Tree water flow. Top: Average monthly water flow at the Big Tree gauge of the San Lorenzo river in Santa Cruz County, CA, plotted on the log-scale. Bottom: Monthly values of Ni\~no 3.4. Both time series span January 1937 through April 2021. }
\label{fig:ex3data}
\end{figure}

For our final example, we consider the average monthly natural water flow (cubic feet per second) at the Big Tree gauge of the San Lorenzo river in Santa Cruz County, CA from January 1937 through April 2021 \citep{btflow}, for a total of 1012 observations. The dataset is included in our package as \code{BTflow}. With this example we show how to build a state-space structure manually (particularly for a dynamic quantile regression), apply our transfer function exDQLM, examine individual components of the estimated quantiles, and perform quantitative model diagnostics to compare models.

For this particular dataset, the interest is in low quantiles as a possible indicator of drought, thus we will consider the $p_0=0.15$ quantile. To investigate any climatological influence on this low quantile, we include the Ni\~no 3.4 index \citep{nino34} as an input covariate in our exDQLM. The Ni\~no 3.4 index is a measure of how anomalously warm or cool the average sea surface temperature is in the Ni\~no 3.4 region ($5^\circ$S-$5^\circ$N, $170^\circ$W-$120^\circ$W) of the Pacific Ocean. It is one of the most commonly used indices to define El Ni\~no and La Ni\~na events. The monthly Ni\~no 3.4 dataset used in our analysis is also included in the package as \code{nino34}. We model the relationship between Ni\~no 3.4 and the quantile with two model structures: a dynamic quantile regression and a transfer function as seen in Section \ref{sec:tf}. To avoid modeling problems caused by the scale and extremes in the Big Tree dataset, we fit the following models to the time series on the log scale seen in Figure \ref{fig:ex3data}.

In addition to the components that capture the relationship with the Ni\~no 3.4 index, we also include a first order polynomial trend component as well as a Fourier form seasonal component that includes a single periodic component with period 12. The Big Tree water flow time series follows a period of twelve months, of which we include only the first (annual) harmonic.
\begin{CodeChunk}
\begin{CodeInput}
R> trend.comp = polytrendMod(1, m0 = 3, C0 = 0.1)
R> seas.comp = seasMod(p = 12, h = 1, C0 = diag(1,2))
R> model = trend.comp + seas.comp
\end{CodeInput}
\end{CodeChunk}

Next, we create the dynamic regression component for our first model using \code{regMod()}. Combining this with the trend and seasonal component we have our complete state-space structure for the dynamic quantile regression model.
\begin{CodeChunk}
\begin{CodeInput}
R> reg.comp = regMod(nino34, m0 = 1, C0 = 1)
R> model.w.reg = model + reg.comp
\end{CodeInput}
\end{CodeChunk}

To allow the Ni\~no 3.4 input components of both models to vary over time, we set the respective discount factors to be $0.95$ (this is done via the parameter \code{df} in the regression fit and the parameter \code{tf.df} in the transfer-function fit). Alternatively, we set the discount factors to 1 and 0.9 for the trend and seasonal components, respectively. This allows the seasonal amplitudes to evolve over time, while any further variation in the $0.15$ quantile from year to year will solely be attributed to the effects of the Ni\~no 3.4 input in our models.

The transfer function model follows the structure detailed in Section \ref{sec:tf}. For this illustration we use \code{exdqlmTransferLDVB()} to obtain a fast approximate fit on the monthly series, although the corresponding exact posterior workflow is also available through \code{exdqlmTransferMCMC()}. On the coarse candidate grid used here, we use the KL divergence as a calibration-oriented screening criterion to select the transfer-function decay rate, and then assess the resulting fitted models with the broader diagnostic comparison reported below. This screen selects a decay rate of 0.85.
\begin{CodeChunk}
\begin{CodeInput}
R> possible.lams = c(0.1,0.3,0.5,0.7,0.85)
R> KLs <- vector("numeric")
R> ref.samp = rnorm(length(BTflow))
R> for(i in 1:length(possible.lams)){
R>   temp.M2 = exdqlmTransferLDVB(y = log(BTflow), p0 = 0.15, model = model,
+                               df = c(1,0.9), dim.df = c(1,2),
+                               X = nino34, tf.df = c(0.95), lam = possible.lams[i],
+                               tf.m0 = c(0,0), tf.C0 = diag(c(0.1,0.005),2),
+                               sig.init = 0.1, gam.init = - 0.1,
+                               tol = 0.05, verbose = FALSE)
R>   temp.check = exdqlmDiagnostics(temp.M2, plot = FALSE,
+                            ref = ref.samp)
R>   KLs = c(KLs,temp.check$m1.KL)
R> }
R> # optimal lambda based off KL divergence
R> possible.lams[which.min(KLs)]
\end{CodeInput}
\begin{CodeOutput}
[1] 0.85
\end{CodeOutput}
\end{CodeChunk}

With these discount factors and optimal $\lambda$, we estimate the dynamic quantile regression model (\code{M1}) and transfer function model (\code{M2}).

\subsection{Static exAL regression on a simulated heteroskedastic example}
\label{sec:ex4static}

To illustrate the static routines under sparse shrinkage, we consider a correlated Gaussian regression benchmark for which the target conditional quantile is known exactly. Let
\[
{\bf x}_i \sim N_8({\bf 0}, \Sigma_x),\qquad (\Sigma_x)_{jk} = 0.5^{|j-k|},
\]
with sparse signal
\[
\boldsymbol{\beta}^\star = (3,\,1.5,\,0,\,0,\,2,\,0,\,0,\,0)^\top.
\]
\aA{We use the following benchmark data-generating mechanism.}
For each target quantile level \(p_0\), we generate
\[
Y_i^{(p_0)} = {\bf x}_i^\top \boldsymbol{\beta}^\star + \sigma_\varepsilon \{Z_i - \Phi^{-1}(p_0)\},
\qquad Z_i \sim N(0,1),\qquad \sigma_\varepsilon = 1.5.
\]
Then
\[
Q_Y(p_0 \mid {\bf x}_i) = {\bf x}_i^\top \boldsymbol{\beta}^\star,
\]
so the true \(p_0\)-quantile is the same sparse linear signal at each fitted quantile level. We use \(n = 160\) training observations and an independent holdout sample of size \(800\), and fit separate static exAL models at \(p_0 \in \{0.05,0.25,0.50\}\). Both fits use the regularized horseshoe (RHS) prior of \cite{nishimura2023shrunken}, implemented in the package through \code{beta_prior = "rhs_ns"}, with \(\tau_0 = 0.15\), fixed slab \(\zeta^2 = 9\), and an unshrunk intercept. We fit a VB approximation with \code{exalStaticLDVB()} using \code{n.samp = 3000}, and an MCMC fit with \code{exalStaticMCMC()} using \code{n.burn = 2000} and \code{n.mcmc = 3000}; the MCMC routine uses its default \code{slice} kernel and is warm-started from the VB fit. The lower-tail VB fit at \(p_0 = 0.05\) converges more slowly, so we allow a larger VB iteration budget there. For reproducibility, we fix the dataset seed at \code{20260714}. Setting \code{dqlm.ind = TRUE} would recover the AL special case discussed in Section \ref{sec:exdqlm}, but here we focus on the general static exAL model under sparse RHS shrinkage.
\begin{CodeChunk}
\begin{CodeInput}
R> Sigma.x = 0.5 ^ as.matrix(dist(1:8))
R> beta.true = c(3, 1.5, 0, 0, 2, 0, 0, 0)
R> rhs.ctrl = list(tau0 = 0.15, a_zeta = 2, b_zeta = 9,
                   zeta2_fixed = 9, shrink_intercept = FALSE)
R> set.seed(20260714)
R> sim.y = function(X.raw, p0) {
      drop(X.raw %*% beta.true) + 1.5 * (rnorm(nrow(X.raw)) - qnorm(p0))
    }
R> X.raw = MASS::mvrnorm(160, mu = rep(0, 8), Sigma = Sigma.x)
R> X = cbind(1, X.raw)
R> p.grid = c(0.05, 0.25, 0.50)
R> y.train = lapply(p.grid, function(p0) sim.y(X.raw, p0))
R> M.ldvb = lapply(p.grid, function(p0) exalStaticLDVB(
+                    y = y.train[[which(p.grid == p0)]], X = X, p0 = p0,
+                    beta_prior = "rhs_ns",
+                    beta_prior_controls = rhs.ctrl,
+                    max_iter = ifelse(p0 == 0.05, 420, 260),
+                    n.samp = 3000,
+                    tol = 1e-4, verbose = FALSE))
R> M.mcmc = lapply(p.grid, function(p0) exalStaticMCMC(
+                    y = y.train[[which(p.grid == p0)]], X = X, p0 = p0,
+                    beta_prior = "rhs_ns",
+                    beta_prior_controls = rhs.ctrl,
+                    n.burn = 2000, n.mcmc = 3000,
+                    init.from.vb = TRUE,
+                    verbose = FALSE))
\end{CodeInput}
\end{CodeChunk}

For static fits, the helper function \code{exalStaticDiagnostics()} summarizes fitted quantiles on a common design matrix and, when a reference quantile is supplied, reports truth-recovery metrics directly. For example, on an independent holdout design:

\section{Conclusion}
\label{sec:summary}

\aA{In this paper we presented the \proglang{R} package \pkg{exdqlm} for Bayesian estimation and analysis of flexible dynamic quantile linear models. The package provides exact posterior inference via MCMC and fast approximate posterior inference via VB for dynamic state-space quantile models, while also supporting transfer-function exDQLMs, static exAL quantile regression with shrinkage priors, forecasting, model diagnostics, and post hoc posterior-predictive synthesis across separately fitted quantiles. Together, these functions provide a practical framework for dynamic and static quantile modeling beyond the AL special case while preserving a computationally convenient latent-variable representation.}

\aA{The examples illustrate the main workflows supported by the package. The dynamic examples show how \pkg{exdqlm} can be used to construct state-space models, estimate dynamic quantiles, compare the exDQLM with its AL special case, examine transfer function effects, and evaluate model fit through diagnostics and forecasting. The static example shows how the same framework can be used for sparse quantile regression with both VB and MCMC, with VB serving as a fast first-pass approximation and MCMC providing the reference analysis when more faithful posterior uncertainty quantification is required. Viewed together, these examples highlight the two software contributions that most clearly distinguish the package within the current ecosystem: Bayesian dynamic state-space quantile modeling and a unified workflow that extends from static regularized exAL regression to post hoc predictive synthesis.}
