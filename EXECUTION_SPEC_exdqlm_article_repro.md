# exdqlm Article Reproducibility Execution Spec

## Status
- Version: `v1` (planning only, no implementation in this spec)
- Scope owner: Antonio Aguirre
- Implementation mode: staged, reproducible, package-first (`exdqlm` APIs only)
- Constraint: do **not** edit `article4.tex` until content selection is finalized

---

## 1) Objective
Build a reproducible article workflow inside `AntonioAPDL/exdqlm---Article` that:
- reuses the strongest ideas from `GSoC-2025/tasks/task_medium`,
- uses **current** `exdqlm` package functions (not standalone `sourceCpp` functions),
- generates publication-quality figures/tables/tests,
- supports clean separation of main-text vs supplementary outputs,
- is deterministic and easy to rerun end-to-end.

## 3) Non-Negotiable Constraints
- No direct `Rcpp::sourceCpp(...)` in article analysis scripts.
- No function calls outside exported `exdqlm` package interface unless explicitly justified.
- No compiled artifacts (`.o`, `.so`) committed to article repo.
- No edits to `article4.tex` until figure/table set is approved.
- Deterministic runs: fixed seeds, fixed parameter grids, fixed output filenames.
- Every final figure/table must be generated by tracked scripts.

## 8) Testing and QA Plan

## 8.1 Script-Level Tests
- Validate parameter domain guards:
  - invalid `sigma`, invalid probabilities, out-of-bounds gamma.
- Validate monotonicity:
  - CDF nondecreasing,
  - quantile nondecreasing.
- Validate inversion quality:
  - `p -> qexal -> pexal` error thresholds over grid.
- Validate special-case identity:
  - `qexal(p0, ...) ≈ mu` scenarios.
- Validate sampling diagnostics:
  - sample quantile checks against target quantiles and tolerances.

## 8.2 Output QA
- Every expected figure path exists.
- Resolution and dimensions are enforced.
- Captions metadata file generated (`analysis/exal/outputs/tables/exal_figure_manifest.csv`).
- No stale outputs from previous runs in final promoted set.

## 8.3 Reproducibility Checks
- Single-command run from repo root.
- Logs include session info and package versions.
- Repeated runs produce identical tables and materially identical plots.

---

## 9) Style and Plotting Standards
- Colorblind-safe palette + grayscale-safe line type differentiation.
- Consistent typography, axis formatting, and legend ordering.
- Publication-safe labels:
  - avoid notation ambiguity (`exAL` naming consistency),
  - explicit parameter values in subtitles or facet strips.
- Fixed seed policy:
  - one global seed for run,
  - optional local seeds for heavy stochastic sections.

---

## 12) Risks and Mitigations
- Numeric instability near gamma bounds:
  - use package `get_gamma_bounds`,
  - avoid evaluating exactly at bounds,
  - add epsilon-margin policy.
- Runtime inflation from dense grids:
  - define standard and high-resolution profiles.
- Drift between exploratory and publication outputs:
  - enforce promote step and manifest file.
- API drift:
  - package version captured in logs,
  - smoke tests for required exported functions.

---

- All Scope A figures and tables generated via tracked scripts.
